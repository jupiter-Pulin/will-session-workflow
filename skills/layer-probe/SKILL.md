# 层探针(layer-probe)

用这个 skill 把「看不懂的 API」变成「一张可复习的保证卡片」。核心纪律:**先让用户猜,再对答案**。用户猜错的位置是唯一的教学点;跳过猜测环节直接讲解,等于毁掉这次学习。

## 抽象层参照系

对答案时统一用这五层定位,不要自创层名:

1. 语言内置(JS 语法、`JSON`、`Promise`)
2. 运行时标准库(Node 的 `fs`/`path`/`process`/`child_process`)
3. 框架·第三方库(NestJS、ORM、redis client)
4. 操作系统内核(syscall、信号、文件系统、进程调度)
5. 硬件(CPU 指令、缓存、磁盘)

## 工作流(严格按顺序)

1. **收窄目标。** 目标必须是单个 API 或单个行为。用户丢来整个文件或多个函数时,先让其挑出「最卡的一个点」,其余留给下次触发。
2. **先猜后讲(阻断点)。** 提出且只提出两个问题,然后停下等待回答:
   - 你猜它属于上面五层中的哪一层?
   - 你猜它向调用者承诺了什么、什么情况下会失败(报什么错)?
   用户回答之前,禁止输出任何解释性内容。用户拒绝猜时,也至少要求其口头选一个层再继续。
3. **对答案,限两层。** 明确指出猜对与猜错的部分。讲解只覆盖:本层的契约是什么 + 下一层的谁提供了这个保证。禁止一路讲到 CPU;更深的层留给用户下次再钻。
4. **给验证实验。** 给出 5 分钟内能跑完的终端命令或 ≤10 行脚本,必须同时覆盖成功路径和失败路径(如触发一次 `EEXIST`)。让用户亲手跑,而不是替他跑;用户主动要求代跑时才代跑。
5. **沉淀卡片。** 向 `/Users/jupiter/ai-experiment/study-knowledge/guarantees.md` 追加一行(文件不存在则按其表头模板重建)。「我当初猜错了什么」一栏不许留空——完全猜对时写「全对:+ 一句当时的判断依据」。
6. **复习钩子。** 追加后数一下卡片总数,每满 10 的倍数,当场出 3 道填空题抽查旧卡片(遮住「契约」和「保证出自哪层」两栏问用户)。

## 卡片格式

```markdown
| 2026-07-05 | fs.mkdirSync 当锁 | 运行时标准库 | 目录已存在则抛 EEXIST,创建是原子的 | OS 内核:POSIX 规定 mkdir 原子 | 不知道 mkdir 是原子操作 |
```

## 红线

- 一次只处理一个 API/行为。
- 第 2 步是阻断点,任何情况下不得先讲后问。
- 讲解不超过两层。
- 卡片当场落盘,不许「稍后补」。
